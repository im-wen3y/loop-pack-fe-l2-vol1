# 검증 및 완료 기준

기본 검증은 정적 검사와 타입 검사까지만 수행한다. 사용자가 정상 동작 또는 런타임 확인을 명시적으로 요청하기 전에는 다음을 임의로 실행하지 않는다.

- 개발 서버 또는 프로덕션 서버 실행
- 브라우저 확인, HTTP 요청, 화면 캡처
- Playwright를 포함한 테스트 실행
- 프로덕션 빌드

"정상 동작 확인", "직접 테스트", "브라우저에서 확인"처럼 명시적으로 요청한 경우에만 런타임 검증을 수행한다. 단순 구현·수정·검토 요청은 런타임 검증 요청으로 해석하지 않는다.

기본 검증 명령어:

```bash
pnpm lint
pnpm exec tsc --noEmit
```

변경 파일 범위만 확인하면 되는 경우 해당 범위에서만 정적 검사를 실행한다. `pnpm test:e2e`, `pnpm build`, `pnpm dev`, 브라우저 기반 검증은 사용자가 요청한 경우에만 실행한다.

인증, 라우팅, SSR·CSR 경계, 브라우저 API처럼 정적 검사만으로 결함 가능성을 충분히 낮추기 어려운 변경은 임의로 런타임 검증하지 않는다. 필요한 확인 항목과 이유를 설명하고 먼저 실행 허가를 요청한다.

화면 관련 코드를 직접 수정했지만 런타임 검증을 수행하지 않은 경우 완료 보고에 다음 문구를 그대로 포함한다.

> 런타임 확인하지 않음

문서, 설정, 서버 전용 코드 등 화면과 무관한 변경에는 이 문구를 넣지 않는다.

# Week 03 — 셀프 리뷰 결과

> 기준: `process.md` 하단 Checklist + 과제 셀프 리뷰 6단계

## 셀프 리뷰 6단계

| #   | 질문                                                                                                                           | 통과 | 비고 |
| --- | ------------------------------------------------------------------------------------------------------------------------------ | ---- | ---- |
| 1   | **레이어** — 컴포넌트 파일만 봐도 화면이 그려지고, Hook 파일만 봐도 기능 흐름이 잡히고, Service 파일만 봐도 API 스펙이 잡히나? | ⬜   |      |
| 2   | **Hook 책임** — 각 Custom Hook을 한 문장으로 설명할 수 있나? "그리고"가 두 번 들어가면 분리 후보                               | ⬜   |      |
| 3   | **상태 3분할** — 서버 상태·클라이언트 상태·파생값을 의식적으로 구분했나? 파생값을 state로 들고 있지 않나?                      | ⬜   |      |
| 4   | **Effect 정당성** — 남은 `useEffect`마다 "외부 시스템과의 동기화인가"라는 질문에 답할 수 있나?                                 | ⬜   |      |
| 5   | **DIP** — Hook이 `axios`·`fetch` 구현에 직접 묶여 있지 않나? 통신 구현이 service에 모여 있나?                                  | ⬜   |      |
| 6   | **분리하지 않은 결정** — "여긴 분리 안 한다"는 결정에도 한 줄 근거가 있나?                                                     | ⬜   |      |

## 관심사 분리

| #   | 항목                                                                   | 통과 | 비고 |
| --- | ---------------------------------------------------------------------- | ---- | ---- |

| 2   | 비즈니스 로직(필터, 검색, 페이지네이션)이 Custom Hook으로 분리되었는가 | ⬜   |      |
| 3   | 포맷팅/유틸 함수가 `utils/`로 분리되었는가                             | ⬜   |      |
| 4   | 컴포넌트는 UI 렌더링에만 집중하는가                                    | ⬜   |      |

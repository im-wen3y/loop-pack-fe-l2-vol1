## 숨은 버그 (참고: `src/productList/README.md`에 상세)

> 과제가 요구한 3개 버그는 전부 수정 완료. self-review 과정에서 추가로 발견한 nit 2건(URL 수동 편집 시에만 발생하는 엣지케이스, 미수정)은 `README.md`의 "추가로 발견한 개선 여지" 섹션 참고.

| 증상                                         | 원인                                        | 처리                                                                                                                                                                                                                      |
| -------------------------------------------- | ------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 새로고침·북마크 시 필터 초기화               | URL에 반영만 하고 초기 마운트 시 읽지 않음  | 수정 — `useProductFilters`/`usePagination`을 `useLocalStorageState`와 같은 lazy initializer 패턴으로 바꿔 마운트 시 URL을 1회 읽어 복원. 라우터가 필요하다던 이전 판단은 근거 과장이었음이 드러나 철회(Playwright로 검증) |
| API 오류 후 전체 새로고침 없이는 재시도 불가 | `onRetry`가 `window.location.reload()` 호출 | 수정 — `refetch()`로 교체, 캐시 유지한 채 재조회                                                                                                                                                                          |
| 검색어 입력마다 API 요청 발생                | debounce 없이 즉시 쿼리 반영                | 수정 — `useDebouncedValue(searchQuery, 300)`로 조회만 debounce, Playwright로 검증                                                                                                                                         |

## 추가한 규칙

> 이번 과제에서 본인 판단으로 CLAUDE.md에 추가한 규칙 (선택)

| 규칙                                                                                                                                       | 근거                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| ------------------------------------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| 데이터 조회 훅은 `fetch`/`axios` 등 통신 구현에 직접 의존하지 않는다 — service 함수만 호출한다 (DIP)                                       | `component-design.md`의 "커스텀 훅은 단일 책임"만으로는 통신 구현이 훅에 스며드는 걸 막지 못함. `useProductListQuery`가 `useGetProductList`(service)만 알고 `fetch`/axios를 모르게 설계한 게 이번 과제의 핵심 판단이었고, API가 계속 늘어날 다음 주 이후에도 반복될 원칙이라 `.claude/rules/component-design.md`의 "상태와 훅" 섹션에 추가함                                                                                                           |
| 동일한 도메인 값 목록(허용 카테고리·정렬 옵션 등)은 `_types` 등 한 곳에 배열로 정의하고 모든 사용처가 참조한다 — 여러 파일에 하드코딩 금지 | URL 초기값 복원 버그를 고치면서 `isCategoryFilter`/`isSortBy`의 검증용 값 목록을 `useProductFilters.ts`에 새로 만들었는데, 이미 같은 값이 `SearchFilter.tsx`(`CATEGORIES`)·`SearchSort.tsx`(`SORT_OPTIONS`)에 있었음 — `code-style.md`의 "기존 코드를 재사용한다" 규칙이 이미 있었는데도 그대로 어김. 기존 규칙이 추상적이라 놓쳤다고 판단해 "같은 값을 여러 파일에 하드코딩하지 않는다"는 더 구체적인 문장으로 `.claude/rules/code-style.md`에 추가함 |
